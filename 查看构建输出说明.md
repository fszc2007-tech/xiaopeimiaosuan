# 查看构建输出说明

## 📋 如何查看 `npx expo run:ios` 的输出

### ⚠️ 重要提示

**后台运行的命令无法直接查看输出**。建议**在前台运行**，这样可以看到实时进度。

---

## 🚀 推荐方式：在前台运行

### Step 1: 打开终端

打开一个新的终端窗口（或使用当前终端）。

### Step 2: 进入项目目录

```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
```

### Step 3: 运行命令（前台运行）

```bash
npx expo run:ios
```

**这样可以直接看到**：
- ✅ 预构建进度
- ✅ CocoaPods 安装进度
- ✅ 编译进度
- ✅ 错误信息（如果有）
- ✅ 模拟器启动状态

---

## 📊 输出示例

运行后，你会看到类似这样的输出：

```
✔ Generated native projects
✔ Installed CocoaPods
✔ Built iOS app
✔ Installed app on simulator
✔ Opening iOS simulator
```

或者如果有错误：

```
✖ Error: CocoaPods not installed
  → Run: sudo gem install cocoapods
```

---

## 🔍 如果已经在后台运行

### 方法 1: 停止并重新运行（推荐）

```bash
# 停止后台进程
pkill -f "expo run:ios"

# 在前台重新运行
cd /Users/gaoxuxu/Desktop/小佩APP/app
npx expo run:ios
```

### 方法 2: 查看进程状态

```bash
# 查看是否在运行
ps aux | grep "expo run:ios" | grep -v grep

# 如果看到进程，说明正在运行
# 但无法直接查看输出
```

## ⏱️ 构建时间参考

| 步骤 | 时间 | 说明 |
|------|------|------|
| 预构建 | 1-2 分钟 | 生成原生项目文件 |
| CocoaPods | 2-5 分钟 | 下载依赖（首次） |
| 编译 | 3-5 分钟 | Xcode 编译 |
| 总计 | 5-10 分钟 | 首次运行 |

**后续运行会更快**（1-2 分钟），因为依赖已缓存。

## ❓ 常见问题

### Q: 为什么看不到输出？

**A**: 因为命令在后台运行。**在前台运行**才能看到输出。

### Q: 如何知道构建是否完成？

**A**: 
- 在前台运行时，终端会显示 "✔ Built iOS app"
- 模拟器会自动打开
- App 会自动安装并启动

### Q: 构建卡住了怎么办？

**A**: 
1. 按 `Ctrl + C` 停止
2. 查看错误信息
3. 根据错误信息修复问题
4. 重新运行
