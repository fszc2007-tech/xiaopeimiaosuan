# 🚀 最简单的解决方案：重启模拟器

## 🔍 问题诊断

- ✅ **基本网络连接正常**（可以 ping 通 IP）
- ❌ **DNS 解析失败**（无法访问域名）
- ❌ **无法访问 Play Store**

这是常见的模拟器网络问题，通常重启模拟器就能解决。

---

## ✅ 推荐解决方案：重启模拟器

### 步骤 1: 关闭当前模拟器

在模拟器窗口中：
- 点击窗口右上角的 **X** 关闭
- 或者运行：
  ```bash
  adb emu kill
  ```

### 步骤 2: 重新启动模拟器

```bash
emulator -avd Medium_Phone_API_36.1
```

### 步骤 3: 等待模拟器完全启动

- 等待看到 Android 主屏幕（1-2 分钟）
- 确保网络连接正常

### 步骤 4: 测试网络

```bash
# 测试 DNS
adb shell ping -c 3 google.com
```

---

## 🎯 如果重启后仍然有问题

### 方法 1: 检查 Android Studio 代理设置

1. 打开 **Android Studio**
2. **Preferences** → **Appearance & Behavior** → **System Settings** → **HTTP Proxy**
3. 选择 **"No proxy"**
4. 点击 **"OK"**
5. 重启模拟器

### 方法 2: 使用手机热点

如果 Wi-Fi 有问题：
1. 打开手机热点
2. Mac 连接到热点
3. 重启模拟器
4. 模拟器会自动使用 Mac 的网络

### 方法 3: 清除模拟器数据

如果问题持续：
```bash
# 删除 AVD 数据（会清除所有数据）
emulator -avd Medium_Phone_API_36.1 -wipe-data
```

**最简单的步骤**:

3. **等待启动完成**
4. **测试网络**: `adb shell ping -c 3 google.com`

---

## 💡 为什么重启有效？

重启模拟器会：
- 重新初始化网络接口
- 重新加载 DNS 配置
- 清除临时的网络问题
