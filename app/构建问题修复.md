# iOS 构建问题修复

## ❌ 遇到的问题

**CocoaPods 安装失败** - 编码问题

错误信息：
```
Unicode Normalization not appropriate for ASCII-8BIT (Encoding::CompatibilityError)
```

## ✅ 解决方案

### 方法 1: 设置环境变量（临时）

```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
npx expo run:ios
```

### 方法 2: 永久设置（推荐）

在 `~/.zshrc` 或 `~/.bash_profile` 中添加：

```bash
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
```

然后重新加载：
```bash
source ~/.zshrc
```

### 方法 3: 使用修复后的脚本

已更新自动构建脚本，会自动设置编码。

## 🚀 重新启动构建

```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
npm run auto:ios
```

## 📝 验证编码设置

```bash
echo $LANG
# 应该输出: en_US.UTF-8
```

## ⚠️ 注意事项

1. **路径问题**: 确保在正确的目录 `/Users/gaoxuxu/Desktop/小佩APP/app`
2. **编码设置**: 必须设置 UTF-8 编码
3. **CocoaPods**: 需要正确安装和配置
