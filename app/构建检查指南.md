# iOS 构建检查指南

## 📍 正确的路径

项目路径是：
```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
```

或者从项目根目录：
```bash
cd ~/Desktop/小佩APP/app
```

## 🔍 检查构建状态

### 方法 1: 检查构建目录
```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
ls -la ios/build/
```

### 方法 2: 检查构建进程
```bash
ps aux | grep xcodebuild
```

### 方法 3: 使用自动脚本
```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
npm run auto:ios:skip-build
```

## 🚀 启动构建

如果构建未开始，运行：

```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
npx expo run:ios
```

或者使用自动脚本（会自动检测并运行测试）：
```bash
cd /Users/gaoxuxu/Desktop/小佩APP/app
npm run auto:ios
```

## ⏱️ 构建时间

- **首次构建**: 15-30 分钟
- **后续构建**: 5-10 分钟

## 📊 构建完成标志

构建完成时会出现：
- ✅ `ios/build/` 目录存在
- ✅ `ios/build/Build/Products/Debug-iphonesimulator/小佩妙算.app` 存在
- ✅ 文件大小约 50-200MB

## 🛠️ 快速命令

```bash
# 进入项目目录
cd /Users/gaoxuxu/Desktop/小佩APP/app

# 检查构建状态
ls -la ios/build/ 2>/dev/null || echo "构建目录不存在"

# 启动构建（如果未开始）
npx expo run:ios

# 使用自动脚本（构建+测试）
npm run auto:ios
```

**提示**: 确保在正确的目录下运行命令！
